# Projects, Pipelines & Saving

Your work in Simple Steps is organized into **projects** and **pipelines**.

---

## Structure

```
your-workspace/
  projects/
    my-analysis/
      sentiment-v1.json      ← a pipeline
      sentiment-v2.json      ← another pipeline
      custom_scoring.py       ← project-specific operations (Tier 3)
    data-cleaning/
      normalize.json
```

- A **project** is a folder inside `projects/`
- A **pipeline** is a `.json` file inside a project folder
- Each pipeline stores the full workflow: steps, formulas, column config, and metadata

---

## Managing Projects

### In the Explorer Sidebar

- **Create a project:** Click the **+** button in the PROJECTS section header
- **Expand a project:** Click the folder to see its pipelines
- **Delete a project:** Click the 🗑 icon (requires two clicks to confirm)

### Saving a Pipeline

1. Click the **💾** icon on the project folder you want to save into
2. Enter a name for the pipeline
3. The current workflow is saved as a `.json` file in that folder

### Loading a Pipeline

Click any pipeline file in the Explorer to load it. The workflow area updates with all the steps, formulas, and data from that pipeline.

### Deleting a Pipeline

Click the 🗑 icon next to a pipeline name. Click again to confirm.

---

## What Gets Saved

A pipeline JSON file contains:

| Field | What It Stores |
|---|---|
| `name` | Display name of the pipeline |
| `steps` | Array of step definitions |
| `steps[].formula` | The formula string (source of truth) |
| `steps[].operation_id` | Which operation this step uses |
| `steps[].config` | Parameter values |
| `steps[].column_config` | Column visibility, width, order |
| `created_at` | When the pipeline was first saved |
| `updated_at` | When it was last modified |

**Data is NOT saved** — only the workflow definition. When you load a pipeline and run it, the data is re-generated by executing each step.

## The File Explorer

The Explorer sidebar also shows a **file tree** of your entire workspace — every file and folder, just like an IDE. This lets you see your project structure, data files, pack source code, and anything else in the directory you launched from.
